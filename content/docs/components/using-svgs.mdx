## Introduction

Working with SVGs in React can be tricky. If you use raw SVGs directly, you cannot easily override props like `width`, `height`, `fill`, or `stroke`.

Manually converting each SVG into a React component is repetitive, time-consuming, and error-prone.

**SVGR** solves this problem by transforming SVG files into fully customizable React components. You can import them directly into your React application and control them using props—such as `width`, `height`, `fill`, `stroke`, `className`, and more.

### Why use SVGR?

Without SVGR, your options are:

1. **Use raw SVGs directly** – You cannot easily override props like `fill`, `stroke`, `width`, or `height`.
2. **Manually convert every SVG into a React component** – Time-consuming, repetitive, and error-prone.

SVGR also integrates **SVGO** for automatic SVG optimization.

<Tabs groupId="configuration" items={['next.config.ts', 'vite.config.ts']}>
  <Tab value="next.config.ts">
    ```ts
    import type { NextConfig } from 'next';

    const nextConfig: NextConfig = {
      turbopack: {
        rules: {
          // 1. SVGs with gradients (keep original fill/stroke)
          './assets/icons/gradients/**/*.svg': {
            loaders: [
              {
                loader: '@svgr/webpack',
                options: {
                  typescript: true,
                  ext: 'tsx',
                  icon: false,
                  svgo: true,
                  svgoConfig: {
                    plugins: [
                      {
                        name: 'preset-default',
                        params: { overrides: { removeViewBox: false } },
                      },
                    ],
                  },
                },
              },
            ],
            as: '*.tsx',
          },

          // 2. All other SVGs – remove fill/stroke
          '*.svg': {
            loaders: [
              {
                loader: '@svgr/webpack',
                options: {
                  typescript: true,
                  ext: 'tsx',
                  icon: false,
                  svgo: true,
                  svgoConfig: {
                    plugins: [
                      { name: 'removeAttrs', params: { attrs: '(fill|stroke)' } },
                      {
                        name: 'preset-default',
                        params: { overrides: { removeViewBox: false } },
                      },
                    ],
                  },
                },
              },
            ],
            as: '*.tsx',
          },
        },
      },
    };
    export default nextConfig;
    ```
  </Tab>
  <Tab value="vite.config.ts">
    ```ts
    import { defineConfig } from 'vite';
    import react from '@vitejs/plugin-react';
    import svgr from 'vite-plugin-svgr';

    export default defineConfig({
      plugins: [
        react({ babel: { plugins: [['babel-plugin-react-compiler']] } }),

        // Gradient icons (keep fills)
        svgr({
          include: ['**/assets/icons/gradients/**/*.svg*'],
          svgrOptions: {
            icon: false,
            svgProps: {},
            plugins: ['@svgr/plugin-svgo', '@svgr/plugin-jsx'],
            svgoConfig: {
              plugins: [
                {
                  name: 'preset-default',
                  params: { overrides: { removeViewBox: false } },
                },
              ],
            },
          },
        }),

        // Normal icons (remove fill/stroke)
        svgr({
          exclude: ['**/assets/icons/gradients/**/*.svg*'],
          svgrOptions: {
            icon: false,
            plugins: ['@svgr/plugin-svgo', '@svgr/plugin-jsx'],
            svgoConfig: {
              plugins: [
                { name: 'removeAttrs', params: { attrs: '(fill|stroke)' } },
                {
                  name: 'preset-default',
                  params: { overrides: { removeViewBox: false } },
                },
              ],
            },
          },
        }),
      ],
    });
    ```
  </Tab>

## Tips & Common Mistakes
When working with SVGs in React/Next.js, there are some common pitfalls and best practices to keep in mind:

1. **ViewBox Removal**

SVGO optimization sometimes removes the `viewBox` property from SVGs.
Without a `viewBox`, SVGs won't scale properly when you change their `width` or `height`.

Solution: Disable removeViewBox in your SVGR configuration (as done in our React/Next.js setup):
```ts
{
  name: 'preset-default',
  params: {
    overrides: {
      removeViewBox: false,
    },
  },
}
```

<br />

2. **Fill/Stroke Not Overridable**

If you try to change the fill or stroke via props and it doesn’t work, it’s likely because the original SVG contains hard-coded attributes.

Solution: Remove them in the configuration:
```ts
{
  name: 'removeAttrs',
  params: {
    attrs: '(fill|stroke)',
  },
}
```
⚠️ Only use this for solid SVGs. Applying it to gradient-based SVGs will turn them into solid colors.

<br />

3. **Avoid replaceAttrValues for Changing SVG Colors**

Using replaceAttrValues only works for the first instance call.
For subsequent renders, the colors won’t update correctly.

Avoid using it for reusable React components.

<br />

4. **Separate Rules for Solid vs Gradient Icons**

Always define two distinct rules in your SVGR/webpack/Vite config:

* Solid icons: remove `fill/stroke` to allow props-based color customization.
* Gradient icons: keep `fill/stroke` intact to preserve the gradient effect.

<br />

5. **Controlling Gradient Colors**

If you want to dynamically control gradient colors via props, convert the SVG into a React component using the SVGR CLI. This gives full control over the component and its styles.

## Using SVGR CLI (Optional)

If you want more control and deeper component customization, you can generate React components from SVGs using the CLI, though in most cases, using the Webpack or Vite plugins is sufficient

<Tabs items={['package.json']}>
  <Tab value="package.json">
  ```json
  {
    ...
    "scripts": {
      ...
      "icons": "npx svgr --typescript --ext tsx -d ./components/icons ./assets"
    }
  }
  ```
  </Tab>
</Tabs>

**Optional SVGO config:**
<Tabs items={['.svgo.yml']}>
  <Tab value=".svgo.yml">
  ```yml
  plugins:
    - name: preset-default
      params:
        overrides:
          removeViewBox: false
  ```
  </Tab>
</Tabs>

Then you can run the script
```bash
npm run icons
```

The generated component will be as follows
<Tabs items={['./components/icons/Android.tsx']}>
  <Tab value="./components/icons/Android.tsx">
  ```tsx
  import * as React from 'react';
  import type { SVGProps } from 'react';

  const SvgAndroid = (props: SVGProps<SVGSVGElement>) => (
    <svg
      xmlns="http://www.w3.org/2000/svg"
      width={48}
      height={48}
      fill="none"
      viewBox="0 0 48 48"
      {...props}>
      <path fill="#000" fillRule="evenodd" d="..." clipRule="evenodd" />
      <path fill="#000" d="..." />
    </svg>
  );

  export default SvgAndroid;
  ```
  </Tab>
</Tabs>

Then you start using it in your components

<Tabs items={['page.tsx']}>
  <Tab value="page.tsx">
  ```tsx
  import AndroidIconComponent from './components/icons/Android';

  const Page = () => {
    return (
      <>
        <AndroidIconComponent />
      </>
    );
  }

  export default Page;
  ```
  </Tab>
</Tabs>
